# Notion風セレクトUIの追加とマスターデータ管理の実装完了

ご要望通り、「場所」と「台の種類」をテキスト形式から、Notion風のセレクトUIに変更し、すべての競技に「場所」を追加しました。

## 変更内容の概要

### 1. マスターデータの導入
- Firestore上に新たに `places` および `machine_types` コレクションを管理するモデルとサービスを作成しました。
- 過去の入力履歴から抽出するのではなく、専用のマスターデータとして保存されるため、名前の編集や削除が一貫して行えるようになりました。

### 2. 全競技への「場所」の追加
- 麻雀、競馬、競艇、オート、競輪の各 `Result` モデルに `place` プロパティを追加しました。
- `EditScreen` にて、全ての競技で「場所」を入力できるように共通のUIフィールドを配置しました。
- メモ入力欄のラベルから「(場所など)」の文言を削除しました。

### 3. Notion風のカスタムセレクトUI (`CreatableAutocomplete`) の実装
- 新しく `CreatableAutocomplete` ウィジェットを作成しました。
- **入力と作成**: 既存のリストに入力文字列が存在しない場合、リストの最下部に「"〇〇" を作成」という項目が表示され、その場で新規マスターデータとして登録されます。
- **編集と削除**: 既存の候補リストの各項目の右側に、鉛筆アイコン（編集）とゴミ箱アイコン（削除）を配置しました。
  - **編集**: アイコンをタップすると編集ダイアログが開き、名前を修正できます。修正はマスターデータに即座に反映されます。
  - **削除**: アイコンをタップすると確認ダイアログが表示され、承諾するとマスターデータから削除されます。

## 動作確認のお願い
アプリを再起動し、以下の動作をご確認ください。
1. 任意の競技の収支追加画面を開き、「場所」欄に入力して新規作成ができること。
2. 作成した場所がサジェストされ、鉛筆・ゴミ箱アイコンで編集・削除が行えること。
3. パチンコの画面では「台の種類」についても同様の機能が動作すること。
4. すべてのカテゴリで場所が入力でき、保存・編集時に正しく値が復元されること。
