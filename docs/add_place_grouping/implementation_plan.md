# 追加要望（UI調整）の実装計画

ユーザーからいただいた以下の3つのUI調整・機能追加を行います。

## 1. 詳細画面での場所ごとの絞り込み（グループ化）
**背景**: 現在、詳細画面（CategoryView）の「グループ」選択肢に「場所」が含まれているのはパチンコのみです。
**提案**: すべての競技（麻雀、競馬、競艇、オートレース、競輪）でも「場所」でグループ化・集計ができるようにします。
- `lib/utils/grouping_helper.dart` の `getGroupableProperties` に、各競技の `place` を追加します。
- `_getPropertyValue` にて、各競技ごとに `place` の値を正しく返すよう処理を追加します。

## 2. 詳細グラフの縦向き化
**背景**: 現在、グラフ画面 (`ChartScreen`) に遷移すると、強制的に画面が横向き (Landscape) に切り替わる仕様になっています。
**提案**:
- `ChartScreen` の画面遷移時に画面を横向きに強制する処理（`SystemChrome.setPreferredOrientations`）を削除し、縦向きのままグラフが表示されるように変更します。
- これにより、スマホを縦に持ったままでグラフを閲覧できるようになります。

## 3. セレクトとキーボードの被りの修正
**背景**: `CreatableAutocomplete`（場所や台の種類の入力欄）のサジェスト一覧を開くと、ソフトウェアキーボードと被って一部が見えなくなってしまう現象が起きています。
**提案**:
- 候補リストの表示領域の高さ制限（`maxHeight`）を固定値ではなく、画面全体の高さやキーボードの高さ (`MediaQuery.of(context).viewInsets.bottom`) を考慮して動的に計算するようにし、キーボードに隠れないように調整します。
- 万が一画面下に十分なスペースがない場合はリストの最大高さを抑え、内部でスクロールできるようにします。
