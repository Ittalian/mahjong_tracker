# 検索（絞り込み）機能の実装計画

収支要素の一番上に検索ボタンを追加し、各プロパティで絞り込みを行えるようにする機能の実装計画です。

## 変更内容

### 1. 検索ボタンの追加
- `lib/widgets/category_view.dart` の一番上（合計収支が表示されているコンテナの右上など）に検索（フィルター）アイコンボタンを追加します。
- このボタンを押すと、絞り込み用のボトムシート（またはダイアログ）が開くようにします。

### 2. 絞り込みUI（ボトムシート）の実装
- 現在のカテゴリに紐づくプロパティ（麻雀であればタイプ、ウマ、レート、場所、メンバーなど）を一覧表示します。
- 登録されている全データから、各プロパティに存在する値のリスト（重複なし）を抽出し、ドロップダウン等で選択できるようにします。
- 「日付（date）」に関しては、月別・年別などの既存の選択肢か、期間指定を行えるようにします。

### 3. 状態管理とフィルタリング処理
- `CategoryView` 内に `Map<String, dynamic> _activeFilters` のような状態を持ち、選択された条件を保持します。
- データ（`results`）を表示用リストや合計金額計算に渡す前に、この `_activeFilters` に基づいてデータをフィルタリングします。
- 絞り込みが有効な場合は、検索ボタンの色を変えるか、「バッジ」を表示して、絞り込み中であることが視覚的にわかるようにします。

## ユーザーへの確認事項（Open Questions）

> [!IMPORTANT]
> 以下の点についてご希望をお知らせください。
> 1. **複数選択について**: 1つのプロパティに対して複数の値を選択できるようにしますか？（例：「場所」で「店舗A」と「店舗B」の両方を含むデータを表示する）それとも、各プロパティにつき1つの値の選択（例：「店舗A」のみ）で十分でしょうか？
> 2. **フリーワード検索について**: プロパティの選択だけでなく、「メモ」などに含まれる自由なテキストでの検索機能も必要でしょうか？
> 3. **UIについて**: 絞り込みはボトムシート（画面下からスライドして出てくるメニュー）の形式で実装してもよろしいでしょうか？

ご確認いただき、問題なければ実装に進みます。
