# 実装完了レポート：スロット分析機能の強化

スロットカテゴリに「合計G数」「RB回数」「BB回数」を追加し、店舗×機種ごとの「大当たり確率」などを複合的に分析できる機能を実装しました。

## 実装内容

### 1. SlotResult モデルと入力フォームの拡張
- `SlotResult` に `totalGames`, `rbCount`, `bbCount` の各フィールドを追加しました。
- 編集画面（`EditScreen`）において、スロットカテゴリが選択された場合のみ、合計G数・RB回数・BB回数を横並びのコンパクトな3列レイアウトで入力できるようにしました。

### 2. ResultCard の情報追加
- 履歴一覧画面（ResultCard）において、スロットの場合は「RB回数」「BB回数」「大当たり確率」を表示するようにしました。
- **大当たり確率の計算**: `合計G数 ÷ (RB回数 + BB回数)` を計算し、`1/X` の形式で表示しています。
- 各入力値が 0（未入力）の場合は `-` と表示し、エラー（0除算など）を防ぐよう対応しました。

### 3. グループ機能（複合）の追加
- 詳細分析のセレクトボックスに、スロット専用の「複合（店舗×機種）」の項目を追加しました。
- `GroupingHelper` を改修し、店舗と機種の組み合わせ（例：`A店 × ジャグラー`）ごとに収支などを集計する機能を追加しました。
- **複合集計ビュー**:
  - 店舗×機種ごとに、合計の収支、G数、RB回数、BB回数を合算します。
  - 合算された数値から、全体としての RB確率、BB確率、大当たり確率（合算値ベース）を計算し、専用のカードUIで一覧表示するように実装しました。

## 検証項目

- [x] ResultCardにRB・BB・大当たり確率が正しく表示されるか（未入力時は `-` 表示になっているか）
- [x] 詳細表示で「複合（店舗×機種）」を選択した場合に、正しいグループ名で合算された結果（収支、G数、確率）が表示されるか
- [x] 静的解析ツール（`flutter analyze`）で今回追加したコード由来のエラーがないか（コンパイルエラーを解消済み）

> [!TIP]
> 既存データは各回数が未入力(`0`)となっているため、確率等は「-」と表示され、複合分析時の合算対象からも自動的に除外される仕様になっています。これにより、過去のデータと混在しても安全に分析が可能です。
