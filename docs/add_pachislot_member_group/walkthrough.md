# パチンコ・スロット共有のグループ機能実装完了

パチンコおよびスロットの収支登録において、一緒に打つメンバー（乗り打ちメンバー等）をグループとして管理し、簡単に入力できる機能を追加しました。

## 実施した内容

### 1. グループ管理の基盤構築
- **`PachinkoGroup` モデル作成**: 麻雀と異なりタイプ（三麻/四麻）の概念を持たないシンプルなモデルを定義しました。
- **`PachinkoGroupService` 作成**: Firestore (`pachinko_groups` コレクション) でのグループの保存、更新、削除を処理します。
- **`GambleMemberService` 作成**: `rxdart` を活用し、過去のパチンコおよびスロットの全収支（`PachinkoService` と `SlotService`）からメンバー名を抽出し、結合・重複排除して候補一覧（ストリーム）を生成するようにしました。

### 2. グループ管理画面の実装
- **`PachinkoGroupEditScreen` 作成**: パチンコ・スロット共有のグループを管理（追加、編集、削除）する画面を作成しました。
- メンバーのドラッグでの並び替え、過去にパチンコまたはスロットで入力したメンバーのサジェスト表示が可能です。

### 3. 収支編集画面の改修
- **`PachinkoEditScreen` / `SlotEditScreen` のアップデート**: 
  - メンバー入力UIを「個人追加」と「グループ選択」で切り替えられるようにしました。
  - すでに保存済みの収支データを編集する際、メンバーリストが保存されているグループのいずれかに合致するかを自動判定し、合致した場合は「グループモード」としてグループ名を表示するロジックを実装しました。
- パチンコについては、「乗り打ち」を選択した時のみメンバー入力UIが表示されるよう制御しています（既存の仕様を維持）。スロットは常にメンバー入力UIが表示されます。

### 4. ホーム画面の連携
- ホーム画面のタブ切り替え時、**「パチ」** または **「スロ」** タブを開いている時に、右下にグループ管理画面へ遷移する `FloatingActionButton` (アイコン: `Icons.groups`) を表示するようにしました。

## 確認事項
全てのコードで静的解析エラー（エラー・ワーニング）が0件であることを確認しています。
実際にアプリをビルドして、ホーム画面から「パチ」「スロ」タブでグループ管理画面が開けること、収支登録でグループからの入力ができることをご確認ください。
