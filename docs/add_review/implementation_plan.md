# レビュー機能の実装計画

## 概要

アプリ内で各ギャンブル（パチンコ、麻雀、競馬、競輪、競艇、オート）の対象（店舗、台、馬、選手など）に対して、複数の評価軸でレビュー（1〜5の星評価）を付けられる機能を追加します。
ホーム画面に「レビュー」ボタンを設置し、専用のレビュー管理画面へ遷移できるようにします。

> [!IMPORTANT]
> 以下の「Open Questions（質問事項）」について、実装前に方針の確認をお願いいたします。

## Open Questions (ユーザーへの確認事項)

1. **「その他の評価軸」の具体的な項目名について**
   競馬や競輪等の「その他の評価軸を考えて実装」について、以下のような項目で固定して実装を進めてもよろしいでしょうか？（もしユーザー自身が自由に項目名を決められる「カスタム評価軸」機能をご希望の場合はお知らせください）
   - **競馬（馬）**: スピード、スタミナ、パワー、根性
   - **競馬（騎手）**: 技術、勝負強さ、ペース配分
   - **競輪・競艇・オート（選手）**: スピード、技術（旋回力など）、メンタル

2. **パチンコ・麻雀の「登録されているリスト」について**
   - 現在Firestoreに保存されている `MachineType` (台) と `Place` (店舗) のデータを読み込んで一覧表示し、各アイテムにレビュー情報を紐づけて保存する仕様でよろしいでしょうか？
   - 「パチンコ以外の場合は要素の追加・削除が可能」とのことですので、パチンコ・麻雀の台・店舗の追加・削除自体はここ（レビュー画面）からは行わず、既存の収支入力画面等のマスタ追加機能に依存する形でよろしいでしょうか？

3. **「レビュー」ボタンの配置場所**
   - ホーム画面（`HomeScreen`）のAppBar右上に「⭐︎ レビュー」のようなアイコンボタンを設置する形でよろしいでしょうか？

4. **競馬などの「追加・削除」の仕様**
   - 競馬や競輪などでは、レビュー画面内で右下の「＋」ボタン等を押して、新しい馬・騎手・選手の名前を入力して追加し、同時に評価を行う（独自のコレクションとしてFirestoreに保存する）仕様でよろしいでしょうか？

## 提案する実装内容（Proposed Changes）

### 1. データモデルの追加 (Models & Services)

Firestoreに新しくレビュー情報を保存するためのモデルクラスと、それを操作するServiceクラスを作成します。

#### パチンコ・麻雀（既存マスタに紐づくレビュー）
- `PachinkoMachineReview` (machineId, 総合評価, 演出, 出玉力, カスタム, 音楽, メモ)
- `PachinkoPlaceReview` (placeId, 総合評価, 回転数, 雰囲気, 店員, メモ)
- `MahjongPlaceReview` (placeId, 総合評価, アクセス, 値段, 雰囲気, 店員, メモ)

#### 競馬・競輪・競艇・オート（独自に追加・削除できる要素）
- `HorseReview` (名前, 総合評価, スピード, スタミナ, パワー, 根性, メモ)
- `JockeyReview` (名前, 総合評価, 技術, 勝負強さ, ペース配分, メモ)
- `RacerReview` (名前, 種別[競輪/競艇/オート], 総合評価, スピード, 技術, メンタル, メモ)

### 2. UIの追加 (Screens & Widgets)

#### [MODIFY] `lib/screens/home_screen.dart`
- AppBarを追加し、右側に「レビュー」画面へ遷移するボタンを配置します。

#### [NEW] `lib/screens/review/review_main_screen.dart`
- レビュー機能のトップ画面。
- 上部にタブ（TabBar: パチンコ、麻雀、競馬、競輪、競艇、オート）を配置し、各ギャンブルのレビューリストを切り替え可能にします。

#### [NEW] `lib/screens/review/components/...`
- **パチンコタブ**: 上部に「台 / 店舗」の切り替えスイッチ (SegmentedButton)。リストにはFirestoreから取得した `MachineType` / `Place` を表示し、タップでレビュー入力ダイアログを表示。
- **競馬タブ**: 上部に「馬 / 騎手」の切り替えスイッチ。FAB（＋ボタン）で新規追加可能。
- 各リストの上部または右上に「絞り込み (Filter)」「並べ替え (Sort)」用のアイコンを設置し、BottomSheetで評価軸ごとの並べ替えやフィルタリングを行えるようにします。

#### [NEW] `lib/widgets/star_rating_input.dart`
- タップで1〜5の星を入力できる共通の評価入力用UIコンポーネントを作成します。

## Verification Plan (検証計画)

1. **Firestore連携の確認**:
   - 各項目のレビューが正しくFirestoreに保存、更新、削除できるか確認。
   - パチンコと麻雀において、既存の台・店舗マスタと正しく紐づいて表示されるか確認。
2. **UI/UXの確認**:
   - 1〜5の星の入力が直感的に行えるか。
   - 評価軸ごとの並べ替え機能（例：「総合評価が高い順」「出玉力が高い順」）が正しく動作するか。
   - 絞り込み機能（例：「総合評価3以上」）が正しく動作するか。
